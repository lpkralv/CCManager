# Config UI Should Reflect Current Configuration State

## What
The app should behave differently when already configured vs unconfigured:

1. **When configured**: Do NOT show the "configure" banner/pane at the top of the dashboard. The app should go straight to the normal view.
2. **Configure dialog**: When clicking the Configure button, the dialog should display the currently configured directory path, making it clear what is already set. The user can then change it if desired.
3. **When unconfigured**: Show the configure prompt at the top as it does now, and the Configure dialog should show an empty/placeholder state inviting the user to set a directory.

## Context
The app is already configured to read projects from the sandbox directory and successfully loads them, but the UI acts as if no configuration exists — showing a configure banner on startup and displaying a blank configure dialog. The UI needs to check the current config state and reflect it accurately.

---
*Source: "the Mac app works. It seems to be already configured to present projects from the 'sandbox' directory. However, the app acts as if it is unconfigured. When I open the app it displays the pane at the top urging me to configure. Also, when I click on the Configure button it does not display the current configured directory, or give any other indication that it is already configured for some directory. It should behave differently (not show the configure pane at the top; display the configured folder in the config dialog) when configured, than when not."*

## Exploration

- **Settings API** (`GET /api/settings`): Returns `effectiveProjectsRoot`, `projectsRootSource` ("env"/"settings"/"none"), and raw settings
- **Frontend state**: `needsSetup = projectsRootSource === 'none'` — only checks explicit config, ignores loaded projects
- **Root cause**: Projects load from `data/inventory.json` (which lists paths), but `settings.json` may not exist and `PROJECTS_ROOT` env var isn't set in Electron. So `projectsRootSource` is "none" even with working projects.
- **Fix needed**: `needsSetup` should also consider whether projects are already loaded. If projects exist, infer the root from their paths. The configure dialog should show the inferred/effective path.
- **Key files**: `public/js/app.js` (loadSettings, needsSetup), `public/index.html` (banner, settings modal), `src/server/routes/settings.ts` (API)
