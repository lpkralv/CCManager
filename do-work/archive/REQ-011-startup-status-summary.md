# Startup Status Summary Table

## What
On startup, both the web dashboard and the MacOS Electron app should display a status summary table listing all tracked projects with: name, status, date of last change, git status, and a brief description of the last change.

## Context
The dashboard currently shows a project list but may not surface all of this information at a glance on load. The `/api/projects/:id/details` endpoint already returns git status and recent commits per project. This request is about presenting a consolidated summary table as the default landing view for both the web app (port 3000) and the Electron app (port 3001).

## Exploration

- **Current dashboard**: 3-column grid (sidebar with project list + dispatch form, center with active tasks + history). No summary table exists.
- **Alpine.js state** (`public/js/app.js`): `projects[]`, `selectedProjects[]`, `activeTasks[]`, `taskHistory[]`. Data loaded via `loadProjects()` and `loadTaskHistory()` on init.
- **API data available**: `/api/projects` returns basic info (name, status, description, metadata.lastModified). `/api/projects/:id/details` returns git status + recent commits but requires per-project call.
- **Need new endpoint**: `/api/projects/summary` to return all projects with git status and latest commit in one call (avoids N+1 problem).
- **CSS patterns**: Dark theme (vars: `--bg-primary`, `--bg-secondary`, `--bg-card`, `--text-primary`, `--accent`), status badges (`.status-active`, `.status-design`, etc.), git indicators (`.git-clean`, `.git-dirty`).
- **Implementation plan**: New backend route → new Alpine.js state + loader → HTML table card at top of main content → CSS table styles matching existing dark theme.
