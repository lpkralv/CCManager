# Improve Failed Task Error Details and Fix Workflow

## What
Enhance failed task entries in the task history to provide sufficient debugging information and enable easy fix requests.

## Context
When tasks fail, the task history entry needs enough information to diagnose and fix the issue.

**Minimum required information:**
- Error codes
- Error messages
- Error locations (file paths, line numbers if applicable)

**Ideal enhancement:**
- Add a button to failed history entries
- Button formats the original prompt and error result information into a prompt
- Sends formatted prompt to CLAUDECODEMANAGER project to be fixed
- This creates a seamless workflow from failure → fix request

## Assets
None

## Triage

**Route: B** - Medium

**Reasoning:** Clear feature request but need to explore how the task history UI renders failed tasks and how the task dispatch system works to wire a "Fix" button.

## Exploration

Key findings:

1. **Task model** (`src/models/task.ts`): Has `error` field (string, optional) that captures error messages. Already stored in history.
2. **Task history card** (`public/index.html:141-159`): Currently shows project name, status badge, truncated prompt, timestamp, and duration. "View Output" button only shows when `task.output` exists. Error details are NOT shown inline on failed cards.
3. **Task output modal** (`public/index.html:356-393`): Already has an error section that shows `task.error` in a pre-formatted block, but only visible when clicking "View Output" (which requires `task.output` to be truthy - failed tasks often have empty output).
4. **Task dispatch** (`public/js/app.js:339-395`): Uses `POST /api/tasks` with `{ projectId, prompt }`. ClaudeCodeManager's projectId is `"claude-code-manager"`.
5. **Button patterns**: `.btn-small.btn-view-output` for action buttons in task cards. Failed tasks use `.task-failed` with red left border.

Key issues to fix:
- "View Output" button hidden when `task.output` is empty, so failed tasks with only `task.error` have no way to see error details
- No inline error preview on failed task cards
- No "Fix" button exists yet

*Generated by Explore agent*

## Implementation Summary

- **`public/index.html`**: Fixed "View Output" button to show when `task.output || task.error` (was hidden for failed tasks with no output). Added inline error preview (`task-error-preview`) below task prompt on failed cards. Added "Fix" button on failed task cards that dispatches a fix request.
- **`public/js/app.js`**: Added `dispatchFixTask(task)` method that formats the original prompt + error details into a structured fix prompt and dispatches it to `claude-code-manager` via `POST /api/tasks`.
- **`public/css/styles.css`**: Added styles for `.btn-fix` (orange/warning button), `.task-header-actions` (button container), and `.task-error-preview` (inline error text with red accent).
