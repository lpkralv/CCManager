# Active Tasks Pane Display Regression

## What
The Active Tasks pane still has display issues when multiple tasks are dispatched:

1. Only one task appears in the pane, even though vertical space is allocated for all of them (suggesting the data is there but rendering fails).
2. Once the first visible task completes, the pane goes blank even though other dispatched tasks are still running.

## Context
This was previously reported and fixed in REQ-008, which addressed a merge-vs-overwrite issue in the WebSocket `initial` message handler in `public/js/app.js`. The symptoms are similar but the bug persists or has regressed. The fix from REQ-008 may not have fully addressed all code paths, or there may be an additional rendering issue (e.g., Alpine.js reactivity not triggering, template iteration bug, or task removal logic clearing the array when one task completes).

## Exploration

Root cause: Alpine.js reactivity breaks when `activeTasks` array is mutated in-place (`.splice()`, `.push()`, `.unshift()`). Alpine tracks array identity — in-place mutations can desync its DOM tracking, causing rendered nodes to go invisible or blank.

Three mutation sites identified in `public/js/app.js`:
1. Task removal: `.splice(idx, 1)` inside setTimeout
2. Initial population: `.forEach(t => push(t))`
3. Task creation: `.push(data.task)`

*Generated by Explore agent*

## Implementation Summary

All three array mutation patterns in `public/js/app.js` changed from in-place mutation to full array replacement:

1. **Task removal** — `.splice(idx, 1)` → `.filter(t => t.id !== id)` (new array)
2. **Task history** — `.unshift()` + `.splice(100)` → `[data.task, ...slice(0, 99)]` (new array)
3. **Initial population** — `.forEach(push)` → `this.activeTasks = [...incomingTasks]` (new array)
4. **Task creation** — `.push(task)` → `this.activeTasks = [...this.activeTasks, task]` (new array)

The `updateTaskInPlace()` method (Object.assign on existing items for output streaming) was left unchanged — property mutation on existing objects is fine for Alpine.js.
