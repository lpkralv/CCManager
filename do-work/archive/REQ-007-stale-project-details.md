# Fix Stale Project Details After Task Completion

## What
After a task completes successfully, the project details modal shows stale/outdated git status information. The data is not refreshed to reflect changes made by the completed task.

## Screenshot 4 Description (Task History)
- TASK HISTORY shows "Vintage Radio" with COMPLETED status
- Task was "commit and push" at 9:03:07 PM, took 32.8s
- Task completed successfully according to the UI

## Screenshot 5 Description (Project Details Modal)
- Modal for "Vintage Radio" project
- GIT STATUS section shows: Branch: master, **5 uncommitted** (in red/orange)
- RECENT COMMITS shows commits up to 1/25/2026 12:30 AM
- No new commit visible from the "commit and push" task that just completed
- Message at bottom: "No do-work queue found for this project"

## Observed Issues
1. Task history shows "commit and push" completed successfully for Vintage Radio
2. Project details modal still shows "5 uncommitted" changes
3. Recent commits list doesn't include the new commit from the completed task
4. Git status data is stale/cached and not refreshed after task completion

## Expected Behavior
- After a task that modifies git state completes, project details should reflect the new state
- Git status should show 0 uncommitted (or current actual count)
- Recent commits should include any new commits made by the task
- Either auto-refresh on task completion, or refresh when modal is opened

## Context
This is a data freshness issue. The project details are likely fetched once and cached, without being invalidated when a task completes that might change the project's git state.

Possible solutions:
- Refresh project details when task completes for that project
- Re-fetch git status when project details modal is opened
- Add a refresh button to the modal
- Invalidate cache on task completion

## Triage

**Route: B** - Medium

**Reasoning:** Clear data freshness bug - project details modal shows cached data. Need to find where modal gets data and add refresh logic when task completes or modal opens.

## Exploration

**`loadProjectDetails(projectId)`** - Fetches fresh data from `/api/projects/:id/details` each time called
**`projectDetails`** - Stores the fetched data, not auto-refreshed
**Task completion handler** - Didn't trigger any refresh of project details

### Fix Strategy

When a task completes/fails/cancels, check if the project details modal is open for the same project, and if so, refresh the data automatically.

*Generated by exploration*

## Implementation Summary

Modified `public/js/app.js`:

**Added auto-refresh logic to task completion handler:**
```javascript
// Refresh project details if modal is open for this project
if (this.showProjectDetails && this.projectDetails?.project?.id === data.task.projectId) {
  this.loadProjectDetails(data.task.projectId);
}
```

When a task completes, fails, or is cancelled:
1. Check if project details modal is open (`showProjectDetails === true`)
2. Check if the modal is showing the same project as the completed task
3. If both true, refresh the project details by calling `loadProjectDetails()`

This ensures git status, commit history, and do-work queue are updated immediately when a task finishes.
