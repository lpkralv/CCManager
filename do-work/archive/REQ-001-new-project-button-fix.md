# Fix NEW Project Button - Missing Init and Description

## What
The NEW button in the PROJECTS pane of the web app creates a new project by cloning CCStartup into the target directory, but it fails to:
1. Store/save the user's project description anywhere
2. Run the project's `/init` command (no git repo, no CLAUDE.md, etc.)

## Context
- User created a new project for "SoapMaking" using the web app
- CCStartup was cloned into the SoapMaking project directory
- No document contains the user's description
- No git repo was initialized
- No CLAUDE.md was created
- The `/init` command was not run

## Requirements
1. Investigate current NEW button implementation
2. Plan fixes to ensure:
   - User's project description is saved/stored appropriately
   - The `/init` command is run after cloning CCStartup
   - Git repo is properly initialized
   - CLAUDE.md is created
3. Verify the plan fully addresses all requirements
4. Implement the fix

## Plan

### Problem Analysis

1. **Current Implementation in `project-creator.ts`**:
   - The code does call `runClaudeInit()` which spawns `claude -p /init --dangerously-skip-permissions`
   - The description is stored in `inventory.json`
   - However, the `/init` skill is **interactive** - it expects to interview the user

2. **Root Cause**:
   - The `/init` command likely failed or timed out because it's interactive
   - CCSTARTUP's .git directory is copied over (not a fresh repo)
   - No mechanism to pass description to `/init`

3. **Architecture Decision**: Create a hybrid approach with fallback:
   - Pre-seed project info before running `/init`
   - Add fallback minimal initialization that doesn't require interaction

### Implementation Steps

1. **Modify `project-creator.ts` to pre-seed description**:
   - Write `.project-info.json` after copying CCSTARTUP
   - Remove CCSTARTUP's `.git` directory before init
   - Initialize a fresh git repository

2. **Modify `runClaudeInit` to pass description**:
   - Pass description as environment variable `PROJECT_DESCRIPTION`
   - Include description in the init prompt

3. **Create fallback `ensureMinimalSetup` function**:
   - Creates basic CLAUDE.md if missing
   - Initializes git if missing
   - Ensures project is usable even if `/init` fails

4. **Update `createProject` flow**:
   - Pre-seed project info
   - Remove CCSTARTUP's .git
   - Run /init with description
   - Call fallback to ensure minimal setup

## Implementation Summary

Modified `src/services/project-creator.ts`:

- `fileExists()` - Check if file exists
- `writeProjectInfo()` - Creates `.project-info.json` with name, description, timestamps

- Now accepts `projectName` and `description` parameters
- Passes `PROJECT_NAME` and `PROJECT_DESCRIPTION` as environment variables

**Modified `createProject()` flow:**

3. Remove template's `.git` directory

5. Run `claude /init` (may timeout - now handled gracefully)
6. Call `ensureMinimalSetup()` to guarantee usable state
7. Add project to inventory

**Key behavior change:** No longer fails if `/init` times out - falls back to minimal setup.
