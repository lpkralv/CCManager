# Investigate and Remove do-work Skill Dependencies

## What
Investigate what features in the CCManager codebase depend on the `/do-work` skill (the request queue system in the `do-work/` folder) and propose workarounds or replacements that do not require that skill. This is important for the clean public release (REQ-028) which strips out all development artifacts.

## Context
The `do-work/` directory and its queue system is a development-time workflow tool, not a user-facing feature. However, some parts of the codebase may reference or depend on it (e.g., the project details endpoint returns do-work queue info, the dashboard may display do-work status). These dependencies need to be identified, and either removed or replaced with equivalent functionality that doesn't rely on the do-work skill infrastructure.
